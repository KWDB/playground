## 🎉 恭喜您，已完成 3.0.0 到 3.1.0 的单机升级演练

您已经按照标准流程完成了一次 KWDB 单机离线升级，掌握了以下关键操作：

- 安装并启动 `3.0.0` 单机实例
- 在升级前准备验证数据并备份数据目录
- 使用 `./deploy.sh upgrade -l` 执行 `3.1.0` 本地升级
- 启动升级后的服务并验证版本与数据

### 升级操作清单

1. 升级前确认服务状态与部署方式
2. 备份用户数据目录 `/var/lib/kaiwudb`
3. 停止 KWDB 服务后再执行升级
4. 升级完成后启动服务并进行数据校验

### 继续实践建议

- 结合生产环境补充端口、磁盘空间和依赖检查
- 将升级前后的 SQL 验证脚本纳入运维巡检流程
- 在真实环境中先进行预演，再安排正式升级窗口

> ⚠️注意：由于内置便捷脚本 `kw-sql` 和 `kw-status` 命令在 `3.1.0` 版本才开始引入，因此在 `3.0.0` 版本中无法使用。但后续版本可以直接使用这两个命令进行数据库操作和状态查询。
